Experiment 2 - Vector Art, Animation, and Interactivity

Imitate

To begin, I had a look at Wes Modes's code that was shown in class. While I would say that it was a very quick pick, I knew from the beginning that I wanted to mess around with Perlin noise. With this in mind, I wrote some starter code to draw a single line that follows the output of p5's noise() function. Following this, I added a way for the wave to change by adding the mouse's y-position as one of the inputs. Then, to try to make the curves in the line more smooth, I tried some methods to "stretch" out the function, by multiplying each point's x-coordinate by a value provided by a slider. At this point, I had also noticed that the first and last points in a curve weren't drawn so I kept that in mind when I moved on to the integration phase.

Integrate

To get the line to appear as if it were moving, I knew it wouldn't be as simple as drawing the line and shifting the coordinates around. Instead, I set up two classes, Line and Point. Point was a simple class, holding the x and y coordinates for organization, while Line did most of the heavy work. To get the line to move, I would have to add multiple points into an array, and then draw a portion of the Points in the array. On its own, this wouldn't move the line, so on each draw cycle I made sure to show a different portion, one that was shifted a few array indexes forward.

Innovate

Finally, to begin the innovation phase, I extended this idea of a line into a Field of lines. I created a field which stored the number of active vertices each line would draw, a starting offset between each line, the x-frequency of the noise function, the spacing in between each line, and a few other parameters. This would allow me to draw a column of lines. I also took some time to refactor a lot of code at this step, modeling my code to more closely fit the OOP paradigm. In refactoring my code, I also managed to forget that I had an extra method call in p5's Draw function, which led to an execution chain with an O(n3) operation time, swiftly crashing my Chrome window by consuming all of the available memory. As I was nearing what I considered to be the finishing point of the experiment, I added a set of controls that would allow any user to change how the lines behave by editing the values in the Field instance created.

Reflection

This was a solo project that focused on playing with Perlin noise. While I had a very small amount of exposure to it in my generative design class over the summer, I don't think I managed to utilize it well until now. In the end I would say that I successfully tamed p5's noise() function to work in the way I would like it to. Overall I was rather hesitant to get started since it took me a long time to understand how to manipulate the output, but once that was over, a lot of creative options felt like they were possible. I think the best part of this was seeing what patterns I could create out of the completed project. I also had some setbacks, as I had crashed p5 two or three times in total, and on some of those occurrences I had forgotten to save hours of work.

One thing I thought about but didn't implement was some code to "push"some of the points in the lines away from the mouse's position, but seeing the crashes from before, I decided against it as it would have been computationally expensive as it would mean modifying the coordinates for affected points within each line in the field.
